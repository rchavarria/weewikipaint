Description

WeeWikiPaint is the web's premier collaborative painting site. This project is part of James Shore's Let's Code: Test-Driven Javascript screencast, a series focused on rigorous, professional Javascript development.

For this project, I've been developing in JavaScript programming language, using NodeJS as the core platform, so JavaScript is in both server and client side, and it's amazing.

I'm using the following NodeJS modules and tools:

  • jake: a tool similar to the old and well-known make, but for JavaScript.
  • http-server: easy to guess, an http server.
  • shelljs: a portable (Windows/Linux/OS X) implementation of Unix shell commands on top of the Node.js API
  • nodeunit: provides easy async unit testing for node.js and the browser.
  • mocha: a simple, flexible, fun JavaScript test framework for node.js and the browser.
  • expect.js: BDD style assertions for node and the browser.
  • phantomjs: An NPM wrapper for PhantomJS, a headless WebKit browser scriptable with a JavaScript API.
  • karma: A simple tool that allows you to execute JavaScript code in multiple real browsers.
  • browserify: browserify will recursively analyze all the require() calls in your app in order to build a bundle you can serve up to the browser in a single script tag.
  • procfile: a module to parse Procfile, needed to deploy on heroku
